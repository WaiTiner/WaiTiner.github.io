Java设计模式

发表于 2018-07-28 | 分类于 java |

创建型

  1. 单例模式：确保一个类只创建一个实例，并提供该类的全局访问点。
  2. 工厂方法：定义一个创建对象的接口，但是由子类决定实例化哪个类。
  3. 抽象工厂：提供一个接口，用于创建相关的对象家族。
  4. 生成器：封装一个对象的构造过程，并允许按步骤创建。
  5. 原型模式：使用原型实例指定要创建对象的类型，然后通过复制该原型来创建对象

行为型

  1. 责任链：使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关 系，将这些对象连成一条链，向他们发送请求，直到有一个对象响应请求为止
  2. 命令：将命令封装成对象中，以便使用命令来参数化其他对象，或者将命令对象放入队列中进行排队，或者将命令对象的操作记录到日志中，以及支持可以撤销的操作
  3. 解释器：为语言创造解释器，通常由语法和语法分析来定义
  4. 迭代器：提供一种顺序访问聚合对象元素的方法，并且不暴露聚合对象的内部；表示
  5. 中介者：集中相关对象之间复杂的沟通的控制方式
  6. 备忘录：在不违反封装的前提下获得对象的内部状态，从而在需要时将对象恢复到最初状态
  7. 观察者：在一个一对多的依赖中，当对象状态发生改变时，它的相应依赖也能相应的收到通知自动更新状态
  8. 状态：允许对象在内部改变时改变它的行为，对象看起来像是修改了它所属的类
  9. 策略：让算法独立于使用它的客户端，定义一系列算法，封装，使他们可以互换
  10. 模板方法：定义一些算法框架，并将一些步骤的实现延迟到子类，通过模板方法，子类可以重新定义算法的某些步骤，而不用改变算法的结构
  11. 访问者：为一个对象结构增加新能力
  12. 空对象：使用什么都不做的空对象来代替null

结构型

  1. 适配器：将一个类接口转换成另一个用户需要的接口
  2. 桥接：将实现和抽象分离出来，使他们可以独立变化
  3. 组合：将对象组合成树形结构来表示”整体/部分”关系，允许用户用相同的方式处理单独对象和组合对象
  4. 装饰者：为对象动态添加功能，添加功能的时候不需要修改代码
  5. 外观：提供一个统一的接口，用来访问子系统的一群接口，使子系统更易使用
  6. 享元：利用共享的方式来支持大量细粒度的对象，这些对象一部分内部状态是相同的
  7. 代理：控制对其他对象的访问
